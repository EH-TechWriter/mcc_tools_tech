Why automation?

  • Tech writing is very repetitious
  • Many tools are not overly WYSIWYG

Publishing FrameMaker

From my memory...

  • Open all files in the book.
  • Update the book. (Fix all the issues)
  • Save as PDF.
  • Finalize the PDF (set document properties, etc.)

Not so bad

That doesn't sound so bad, until...

  • You have a book with 20+ chapters.
  • You forgot to tweak a condition setting.
  • There's some other last minute change

Maybe it is that bad

  • You have to make sure your laptop
    • Has the correct version of software.
    • Has the correct version of templates.
  • You follow the 6-20 step procedure for each document

Enter automation

Take advantage of computers.

Computers are designed to do repetative tasks.

What automation can do

  • Start applications, processes, scripts
  • Move files
  • Copy files
  • Zip and unzip files
  • And so much more...

Build system advantages

  • Everyone is using the same set of templates and software
  • All builds are the same each time
  • Can be scheduled/run automagically

Can include quality control

You can automate more than just making things and putting them places.

  • Spelling and grammar checks (https://www.acrolinx.com/)
  • Link checking
  • Accessibility
  • Source validation
  • Log checking

What can a script do?

Yes. It can do that. (If you know how to code it)

There are many different scripting languages. Different languages are better at different things.

  • Manipulate files and folders
  • Manipulate strings
  • Write to files

For example

  1. Download latest and greatest files from source control.
  2. Copy files and folders around where you need them.
  3. Start a build/manipulation of your documentation.
  4. Zip up build results.
  5. Copy files to publication location.

A few scripting languages

This is just a smattering. They keep inventing new ones.

  • Shell scripting - native UNIX. Command line put into files.
  • Batch scripting - native Windows/DOS. Command line put into files.
  • Power shell - The new Windows scripting language.
  • Perl - Stupid good at Regular Expressions. Very lazy.
  • Python - The scripting language I keep meaning to learn.
