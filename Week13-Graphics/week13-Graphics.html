  • Animated content

Image types

Use these types of images in your documentation.

Raster images

JPG, PNG, GIF, BMP

Stores information pixel by pixel, color & position, with some compression algorithm

Vector images

SVG

Stores information as shapes. Can display at any size with same detail

More about raster images

  • BMP — Windows only. Low quality. Avoid.
  • GIF — Supports transparency and animation. Used to be proprietary. Good for line art.
  • JPG — Great for photographs. Not so great for diagrams.
  • PNG — Supports transparency. Decent for screenshots, diagrams, and photos.

Which type to use

Depends on your output.

Today, PNG is the most universal.

Decent quality for web or PDF, suports transparency, works for screenshots, diagrams, or simple photos.

If you need animation, use GIF.

For actual photographs, use JPG.

SVG can work any where at any resolution. Browser support is recent.

For print

Do more research than I'm giving you

Print needs high quality images, at least 600 dpi.

All diagrams should be vector (or combo) images, such as EPS.

Proprietary image formats

I also call these source files. In general, don't reference from documents as most transformation tools can't handle them.

  • AI — Adobe Illustrator
  • CDR — Corel Draw
  • PSD — Photoshop
  • SNAG — Snag-it
  • VSD(X) — Visio

But wait there's more...

There are dozens of other file formats out there.

For example, super high resolution cameras give you RAW files. I don't know what CAD programs create.

There are many other image editing tools out there.

Use what you know, and what your company can afford.

Be familiar with how to work with raster images and how to work with vector images.

Raster Image editing tools

  • Microsoft Paint — Okay for simple edits, but edits as a bitmap (raster).
  • TechSmith Snag-it — Screenshot tool with basic editing features. Fair decent.
  • Adobe Photoshop — Great for image and photos. Expensive and often overkill.
  • Corel PhotoPaint — Great for image and photos. Expensive and often overkill.
  • Gimp — Open Source and Free.

Vector image editing tools

  • Adobe Illustrator — Need a degree in Graphic Design to use. Expensive and overkill.
  • CorelDraw — Need a degree in Graphic Design to use. Expensive and overkill.
  • Microsoft Visio — Fantasic for diagrams. Expensive but fabulous. (Also does DB diagrams)
  • InkScape — Opensource and Free.
  • Dia — Opensource and free. Quirky, poor man's Visio

Graphic terms

A few concepts you should know

Canvas size — the available space available to draw in.

DPI — Dots per Inch. How many pixels display in an inch. Used to define the resolution.

Image size — usually pixels by pixels. The physical size on screen or page will depend on the resolution of the image.

Layer — a container in a graphic for a set of objects

Path — a fancy "line" in a vector application

Colors

RGB — Colors defined by red, green, and blue levels. Used with light, i.e. monitors.

CYMK — Colors defined by cyan, yellow, magenta, and black. Used with print.

For most of tech writing, use RGB colors. They are often defined using Hex codes.

Don't use too many colors.

Make sure your colors are accessible.

Follow any marketing/corporate guidelines.

Editing an image

Tool-agnostic guidelines

  • If at all possible, avoid text in your images. Saves on translation costs.
  • If you must have text, try to use a tool with layers and put all text on the same layer.

More image editing

If you have callouts, text, highlighting, blurring, etc.:
  • Reference the source screenshot or have it on a separate layer
  • Do the work in a graphics tool that you can save without "compressing layers"
  • Save the source in the proprietary format and in the output you need

Saving your images

If it's just an icon or screenshot, save as the output format (PNG)

If you are making any sort of changes, save as both a source file such as AI and your output PNG

Usually, save both files with the same name. Exception: If you need different resolutions of the same image for different outputs.

Set the image size and resolution when exporting/saving as the output format.

In general, monitors are set to 96 DPI. (ish)
